# 单片机教程

单片机使用外设流程：开启时钟，配置寄存器，启动外设，使用

| 操作       | 写法               |
| ---------- | ------------------ |
| 设置某一位 | 寄存器 \|= (1<<位) |
| 清除某一位 | 寄存器 &= ~(1<<位) |



## RCC寄存器（时钟使能寄存器）

RCC可以控制单片机上不同外设的时钟开关，如何GPIO等

## GPIO引脚与GPIO端口的关系

&emsp;单片机上有不同**GPIO引脚** :**PA0-PA15**&ensp;**PB0-PB15** &ensp;**PC0-PC15**，每16个引脚组成一个GPIO端口GPIOA、GPIOB、GPIOC，然后每一个GIPO端口都有几个重要的寄存器**CRL**、**CRH**、**IDR**、**ODR** 

| 寄存器                                 | 名称           | 作用             | 配置引脚 |
| -------------------------------------- | -------------- | ---------------- | -------- |
| **CRL**（Configuration Register Low）  | 低位配置寄存器 | 配置引脚模式     | PA0-PA7  |
| **CRH**（Configuration Register High） | 高位配置寄存器 | 配置引脚模式     | PA8-PA15 |
| **IDR**（Input Data Register）         | 输入数据寄存器 | 读取引脚输入电平 | /        |
| **ODR**(Out Data Register)             | 输出数据寄存器 | 控制引脚输出电平 | /        |

&emsp;配置寄存器每4位（**CNF1,CNF0,MODE1,MODE0**）配置一个引脚，最多可以控制8个引脚，所以配置寄存器是**4\*8=32位**

&emsp;输入输出寄存器也都是32位，但是只有低16位可用

## 每个引脚常见工作模式

| 模式名称     | IO   | 表现形式                                       |
| ------------ | ---- | ---------------------------------------------- |
| **输入浮空** | 输入 | 外设输入什么，它就输入什么，没有接设备则随机01 |
| **输入下拉** | 输入 | 默认输入0，外设拉高时变1                       |
| **输入上拉** | 输入 | 默认输入1，外设拉低时变0                       |
| **模拟输入** | 输入 | 采集模拟信号                                   |
| **开漏输出** | 输出 | 只能输出0，输出1时                             |
| **推挽输出** | 输出 | 能主动输出1，能主动输出0                       |

## 根据GPIO模式推断CNF和MODE配置（输出）

### 确定MODE

* 确认输入还是输出：输入->MODE = 00、输出->MODE ≠ 00
* 若为输出确认输出速度：01【10MZ】10【2MZ】11【50MZ】 确定CNF

### 确定CNF

* 若为输出

| 模式     | CNF  |
| -------- | ---- |
| 推挽输出 | 00   |
| 开漏输出 | 01   |
| 复用推挽 | 10   |
| 复用开漏 | 11   |

* 若为输入

| 模式          | CNF  |
| ------------- | ---- |
| 模拟输入      | 00   |
| 浮空输入      | 01   |
| 上拉/下拉输入 | 10   |

# GPIO输出寄存器

## ODR（输出数据寄存器）

因为一个GPIO端口只有16个引脚，但是ODR寄存器有32位，所以高16位无法使用

## BSRR（GPIO置位/复位寄存器）

BSRR与ODR不同，BSRR直接由硬件操作，不需要像ODR一样要CPU进行读改写操作修改寄存器的值，BSRR可以硬件直接修改对应引脚的输出，不需要CPU干预。

BSRR也是32位，其中低16位为控制置1，高16位为控制清零，分别对应16个GPIO引脚，有时会出现BRR寄存器，实际就是将BSRR的高16位从新命名了，与BSRR加以区分

# GPIO输入寄存器

## IDR寄存器（输入数据寄存器）

IDR也是一共32位，只有低16位有效，用于控制0~16号引脚的输入，高16位保留
